0.1 Data Types

Strings (or character class) are one of the basic data types (classes) in R.

Other data types include:

  1. Logical (TRUE/FALSE)
  2. integer (0/1/2/3/4)
  3. numeric (3.2, -7.14, 11.78)
  4. factor (red/blue/green) with variants:
  • ordinal factor (low/medium/high)
  • factor with labels (1/2, labels = c(“female”, “male”))
  1. complex (3 + 2i, 7 - 3i)
  2. (and of course) strings (“Bob”, “yellow”, “3.14”)

anything can be a string - so it is the default when reading in data of mixed types.

0.2 A first string vector

Here is an example of a vector of six strings. Execute this chunk by clicking the blue Run code button (top right) within the chunk to see the strings object, its length, and its underlying class.

This gives you three sets of results

  1. the vector of 6 strings, with index numbers in brackets on the left.
  2. the length of the strings object (6)
  3. the class of the strings object (character) Note that each of these results are vectors (even if only of length 1), and are numbered.

0.3 The Good and the Bad of Coercion

Variables in a data frame in R are all vectors. Each column represents a vector.

In a vector, all data elements have to be of the same data type (class). If they are not, they will be “coerced” to a common data type or class. This can be convenient at times (you always get a viable vector of a single class), but can also be dangerous if you don’t see it coming (because it happens silently, behind the scenes), and can result in downstream errors later.
Below are some illustrative examples of mixed data in a single vector and the resulting coercion of data classes.

Predict what you think the classes of the vectors values and numbers will be (logical, numeric, character??) in the chunk below. Then run each of the code chunks below with the Run code button. Check your predictions. See what happens when you try to multiply these vectors.

values <- c(TRUE, FALSE, 7)
values
class(values)
pi * values^2 
numbers <- c(3, 9, " ")
numbers
class(numbers)
numbers <- c(3, 9, " ")
numbers * 2

Note that you can multiply TRUE and FALSE values (in values because these will be coerced to 1 and 0 when in a vector with a numeric), and an extra space in a column (often seen in data imported from Excel) can cause coercion of a numeric variable to the character class (as seen in numbers), making attempts to multiply or otherwise numerically manipulate this variable throw an error like Error in numbers * 2 : non-numeric argument to binary operator.

0.5 Finding Patterns in Strings

0.5.1 str_detect()

Let’s start with the strings vector from above use str_detect() to find “alc” and “cirr” in the vector. str_detect() has 2 arguments, string (the string to be searched), and pattern (the pattern to search for). These two arguments are common to most of the stringr functions. Run the code block below for an example

strings <- c("alcoholic cirrhosis", "cirrhotic", "alcoholism", "alcohol abuse disorder", "alcohol toxicity", "alcoholic hepatitis")
str_detect(strings, "alc")
str_detect(strings, "cirr")

This is somehat helpful to get a logical vector of TRUE and FALSE back (this will turn out to be more helpful later).

0.6 Subsetting Text

0.6.1 str_sub()

Now use str_sub() to subset bits of the text. The arguments of str_sub() are str_sub(text, start_integer, end_integer) Try both positive and negative (small) integers. See if you can figure out how the negative numbers work.

strings <- c("alcoholic cirrhosis", "cirrhotic", "alcoholism", "alcohol abuse disorder", "alcohol toxicity", "alcoholic hepatitis")
strings
strings <- c("alcoholic cirrhosis", "cirrhotic", "alcoholism", "alcohol abuse disorder", "alcohol toxicity", "alcoholic hepatitis")
strings %>% 
  str_sub(3,7)

strings %>% 
  str_sub(4,-3) #negative numbers count backward from the end of the string.

strings %>% 
  str_sub(1,10)

0.7 Measuring and Changing String Length

0.7.1 str_length()

Sometimes you want to measure or control the length of strings try str_length() with strings, and with mrns (medical record numbers). str_length() only needs one argument, string.

strings <- c("alcoholic cirrhosis", "cirrhotic", "alcoholism", "alcohol abuse disorder", "alcohol toxicity", "alcoholic hepatitis")
mrns <- c("012345678", "23456789     ", "345678901   ", "123456")
strings <- c("alcoholic cirrhosis", "cirrhotic", "alcoholism", "alcohol abuse disorder", "alcohol toxicity", "alcoholic hepatitis")
mrns <- c("012345678", "23456789     ", "345678901   ", "123456")

str_length(strings)
str_length(mrns)

0.7.2 str_trunc()

when the strings are too long, you can truncate them with str_trunc(string, length) Try this with strings and mrns below to get them all to length 9.

strings <- c("alcoholic cirrhosis", "cirrhotic", "alcoholism", "alcohol abuse disorder", "alcohol toxicity", "alcoholic hepatitis")
mrns <- c("012345678", "23456789     ", "345678901   ", "123456")
strings <- c("alcoholic cirrhosis", "cirrhotic", "alcoholism", "alcohol abuse disorder", "alcohol toxicity", "alcoholic hepatitis")
mrns <- c("012345678", "23456789     ", "345678901   ", "123456")

str_trunc(strings, 9)
str_trunc(mrns, 9)

Notice how the ellipsis is used to show that something has been cut off.

0.7.3 str_pad()

Sometimes leading zeros get left off of numbers, especially when they have been passed through Excel (which by default strips off leading zeroes from numbers, or with MRNs. You can pad these to the right length with str_pad(text, length) This defaults to padding with spaces on the left. You can add a side argument for “right”, “both”, or “left” to str_pad i.e. side = “left” and a pad argument for what character to pad with i.e. pad = “0” Try to get all the MRNs to length 9, with the correct padding character on the correct side. Note that you have to trim off whitespaces first

mrns <- c("012345678", "23456789     ", "345678901   ", "123456")
mrns <- c("012345678", "23456789     ", "345678901   ", "123456")
mrns %>% 
  str_trim() %>% 
  str_pad(side = "left", width = 9, pad ="0")

0.7.4 str_trim()

If you just want to trim off extra whitespaces, you can use str_trim(string) Try this with mrns 
mrns <- c("012345678", "23456789     ", "345678901   ", "123456")
mrns
str_trim(mrns)

0.10 Combining and Splitting Strings

0.10.1 Combining Strings with str_c()

Sometimes you get strings as a vector of string chunks, but you actually want them as a single character string. This can be fixed with str_c() for combine (or collapse) Try the chunk below, then change the collapse character to three spaces, or space-underscore-space, or just a single space.

strings <- c("alcoholic cirrhosis", "cirrhotic", "alcoholism", "alcohol abuse disorder", "alcohol toxicity", "alcoholic hepatitis")
str_c(strings, collapse = "+")

0.10.2 Splitting Strings with str_split()

Sometimes you get strings that combine two data fields, but you actually want them separated into different variables. Common examples include Name (combines firstname and lastname) and Month-Day, which contains two distinct variables. This can be fixed with str_split(). str_split() has two main arguments, string and pattern. Review the small bdays dataframe in the chunk below, then split the two variables using str_split() on the patterns space " " or dash “-” by running the code chunk.

bdays <- data.frame(Name = c("Bob Anderson", "Cindy Davis", "Eric Fields", "Gemma Harris"),
           Birthday = c("Jan-22", "May-14", "Apr-04", "Dec-11"))
bdays$Name
bdays$Birthday  %>% str_split("-")

0.10.3 Splitting strings to separate variables with separate()

The separate() function is in the tidyr package, because it is so often needed in tidying datasets. It conveniently splits variables that combine data into distinct new variables. separate() has 4 main arguments: data, col (column to split), into (names of new resulting columns), and sep (which defines the separator to split on). Run the example below for the Name column, then write code for a similar separate function on the Birthday column in the code chunk below. Try to end up with 4 distinct columns for the bdays dataset.

bdays %>% separate(col = Name, into = c("firstname", "lastname"), sep=" ")
bdays %>% 
  separate(col = Name, into = c("firstname", "lastname"), sep=" ") %>% 
  separate(col = Birthday, into = c("month", "day"), sep="-")
